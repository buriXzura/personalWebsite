File System Metadata Snapshotting

Comprehensive Cross-Platform Data Recovery Solution
Systems Programming • Data Recovery • File System Internals

Executive Summary

This project implements a comprehensive file system metadata snapshotting solution for two major production file systems: EXT4 (Linux) and VFAT/FAT32 (Windows). The system creates lightweight snapshots of file system metadata structures that can be used for data recovery even when the original storage device or file system becomes corrupted.

Core Innovation: Extracting and preserving critical metadata structures in a compact, custom format that maintains file system hierarchy and block mapping information without storing actual file content, making recovery operations feasible even from partially damaged storage devices.

🚀 Key Features & Capabilities

💾 Cross-Platform Support

Native support for both EXT4 (Linux) and VFAT/FAT32 (Windows) file systems with deep understanding of internal structures.

⚡ Lightweight Snapshots

Metadata-only storage approach creates compact snapshots without duplicating file content, enabling efficient backup operations.

🔧 Direct Block Access

Bypasses kernel file system layer using direct block device access, enabling recovery from unmountable or corrupted file systems.

🗂️ Complete Hierarchy Preservation

Maintains full directory structure and file organization, preserving parent-child relationships for accurate recovery.

🔍 Advanced Filtering System

Sophisticated search capabilities with time-based, size, permission, and pattern-matching filters for efficient data location.

💻 Shell Interface

Unix-like command interface for navigation, file listing, extraction, and tree visualization of recovered file systems.

🏗️ Architecture & Implementation

🐧 EXT4 Implementation

  • Superblock structure analysis and extraction
  • Group descriptor table processing (32-bit & 64-bit)
  • Inode structure parsing with extent tree support
  • Directory entry extraction and hierarchy building
  • Extent-based and traditional block addressing
  • Support for large file systems (>2TB)

🪟 VFAT/FAT32 Implementation

  • Boot sector and BPB (BIOS Parameter Block) analysis
  • FAT table processing and cluster chain following
  • Directory entry parsing with 8.3 format support
  • Long filename (LFN) Unicode handling
  • Cluster mapping and file reconstruction
  • Deleted file recovery capabilities

📊 Performance Characteristics

EXT4 Complexity
O(n) time complexity where n is the number of files and directories. Efficient streaming processing for large file systems.
Memory Efficiency
Streaming algorithms prevent excessive memory usage, supporting file systems with millions of files.
Storage Overhead
~176 bytes per file for EXT4, ~36+ bytes per file for VFAT, plus fixed overhead for file system structures.
Recovery Speed
Linear search O(n) for files, O(log n) for hierarchical navigation, with parallel extraction capabilities.
